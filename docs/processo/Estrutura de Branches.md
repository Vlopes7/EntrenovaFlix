# Estrutura de Branches e Processo de Pull Request

Descrição do fluxo de trabalho adotado pela equipe para organização de branches, prazos e revisão de código.

---

## Estrutura das Branches

- **`main`**: Branch principal e estável, representando a versão em produção.
- **`feature/<id-da-story>-descricao-curta`**: Utilizada para o desenvolvimento de funcionalidades com base em User Stories.

> Exemplo: `feature/US123-login-page`, `feature/US145-dashboard-filter`

---

## Processo de Code Review

- Todo código deve passar por **revisão de pelo menos um desenvolvedor** antes de ser aprovado para a branch `main`.
- O revisor deve fornecer **feedback claro e detalhado** para o desenvolvedor responsável pela task.
- O **autor da task** é responsável por testar e corrigir seu código.

---
  
## Checklist antes de abrir um PR:

- [ ] Código foi revisado
- [ ] Testes foram executados
- [ ] Documentação foi atualizada

---

## Tempo de aprovação

- O tempo médio esperado para a aprovação de um PR é de **até 24 horas**.
- **Qualquer desenvolvedor pode aprovar um PR**.
